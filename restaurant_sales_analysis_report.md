# Technical Specification & Architecture Report: Restaurant Sales Analysis

This document details the system design, database configurations, analytics pipelines, and frontend layouts for the **Restaurant Sales Analysis** application.

---

## 1. Executive Summary

The Restaurant Sales Analysis application is a Single Page Application (SPA) designed to parse, aggregate, and visualize transaction records. It operates on a synthetic CSV database generated by a Flask backend and presents real-time KPI metrics and animated ApexCharts in a custom dark-mode glassmorphic interface.

---

## 2. Technology Stack

The details of the system stack:

| Layer | Component | Version / Standard | Purpose |
| :--- | :--- | :--- | :--- |
| **Backend & APIs** | Flask | `3.0.3` | Handles template routing, serves REST APIs, and serves static files |
| **Data Engine** | Pandas | Core library | Reads transaction records from the CSV file and calculates metrics |
| **Templates** | HTML5 | Modern standard | Renders visual grids and hooks JS actions |
| **Client Script** | Vanilla JS | Modern ES6 | Fetches data asynchronously, manages filters, and loads charts |
| **Graphics Engine** | ApexCharts | Vector-based SVG | Renders area charts, donut charts, and horizontal bar charts |
| **Stylesheets** | Vanilla CSS3 | Modern CSS | Injects responsive grid layout and dark amber neon overrides |

---

## 3. Data Schema & CSV Structure

The transaction records are saved locally at:
`projects/resturant/restaurant_sales_data.csv`

The database contains 500 rows structured as follows:

| Column | Data Type | Description |
| :--- | :--- | :--- |
| **Transaction_ID** | String | Unique index identifier (e.g. `TXN-1001` to `TXN-1500`) |
| **Date** | Date | Sales date over a 30-day range (`YYYY-MM-DD`) |
| **Time_Of_Day** | String | Classification of meal session (`Lunch`, `Dinner`, `Snack`) |
| **Item_Name** | String | Product name (e.g., Pizza, Burger, Pasta, Salad, Cake) |
| **Category** | String | Item group classification (`Mains`, `Appetizers`, `Desserts`, `Beverages`) |
| **Quantity** | Integer | Number of units purchased ($[1, 4]$) |
| **Unit_Price** | Float | Fixed cost per item |
| **Total_Amount** | Float | Computed total invoice cost ($\text{Quantity} \times \text{Unit\_Price}$) |
| **Payment_Method** | String | Settlement method (`Cash`, `Credit Card`, `Mobile Wallet`) |
| **Rating** | Integer | Customer rating metrics ($[3, 5]$ stars) |

## 4. System Data Flow

The flow of analytics inside the application:

```text
[Check CSV file] ──> [If missing: generate and save 500 rows to disk] 
                             │
                             ▼
[Flask Server runs] ──> [Deliver index.html & static stylesheets]
                             │
                             ▼
[AJAX Fetch calls] ──> [Pandas aggregates CSV data] ──> [Send JSON response]
                             │
                             ▼
[Render client-side] ──> [Display KPIs, filter logs, and draw ApexCharts]
```

## 5. UI Style System

- **Dashboard Layout**: Split into summary KPI cards, chart sections (area chart + donut chart), and details section (popular items horizontal bar + transaction table).
- **Color Accent Palette**:
  - Main background: `#0B0E14` (Deep dark)
  - Neon Amber highlight: `#FF9F00` (representing food theme warmth)
  - Neon Cyan highlight: `#00F2FF`
  - Neon Green highlight: `#39FF14`
  - Gold Accent: `#FFD700`
